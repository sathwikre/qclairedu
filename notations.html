Section 01

What Are Asymptotic Notations?

Mathematical tools to describe how algorithms scale as input size grows toward infinity — ignoring constants and focusing purely on growth rates.

O
Big O
Upper bound — worst case ceiling
Ω
Big Omega
Lower bound — best case floor
Θ
Big Theta
Tight bound — exact growth rate
o
Little o
Strict upper — f/g → 0
ω
Little ω
Strict lower — f/g → ∞
🌍 Real-World Analogies
Big O — Worst Case
"Worst-case traffic estimate. I'll arrive at most in 2 hours."
Big Ω — Best Case
"I'll always spend at least 20 minutes cooking this meal."
Big Θ — Exact Rate
"The project takes exactly proportional to team size."

O

Big O Notation

Upper Bound — the algorithm will never be worse than this. It's a ceiling on growth.

f(n) ≤ c · g(n)    for all n ≥ n₀
Upper Bound Mode — Visualizing f(n) = n² + n ∈ O(n²)
Constant c (upper)
c=1 c=2 c=5
n₀ threshold
1 n₀=5 30
✓ Valid Upper Bound
f(n) ≤ 2·n² for all n ≥ 5
📐
Upper Bound
f(n) can never exceed c·g(n) for large enough n. g(n) acts as an asymptotic ceiling.
🔵
Worst Case
Used for worst-case guarantees. If O(n²), it will never be slower than quadratic.
✂️
Drop Constants
O(3n²) = O(n²). Constants and lower-order terms vanish as n → ∞.

Ω

Big Omega Notation

Lower Bound — the algorithm will never be better than this. It's a floor on performance.

f(n) ≥ c · g(n)    for all n ≥ n₀
Lower Bound Mode — f(n) = n² + n ∈ Ω(n²)
Constant c (lower)
0.1 c=0.8 2
n₀ threshold
1 n₀=3 30
✓ Valid Lower Bound
f(n) ≥ 0.8·n² for all n ≥ 3
📏
Lower Bound
f(n) is always at least c·g(n) for large n. g(n) is an asymptotic floor.
🟢
Best Case
Used for best-case guarantees. Ω(n) means it can never be faster than linear.
🔄
Dual of Big O
f ∈ O(g) iff g ∈ Ω(f). They are mirror images of each other.

Θ

Big Theta Notation

Tight Bound — f(n) is sandwiched between two scaled versions of g(n). The most precise notation.

c₁·g(n) ≤ f(n) ≤ c₂·g(n)    for all n ≥ n₀
Tight Bound Mode — f(n) = n² + n ∈ Θ(n²) — trapped between c₁n² and c₂n²
c₁ (lower constant)
c₁ = 0.7
c₂ (upper constant)
c₂ = 2.0
n₀ threshold
n₀ = 4
✓ Tight Bound Holds
0.7n² ≤ f(n) ≤ 2n² for n ≥ 4
🎯
Tight Bound
f = Θ(g) means g perfectly captures the growth rate — not just a bound, but the exact class.
🔗
Both Directions
f ∈ Θ(g) ⟺ f ∈ O(g) AND f ∈ Ω(g). It's the intersection of both.
⚡
Average Case
Θ notation is often used to describe average-case behavior with precision.

Little o & Little ω

Strict asymptotic bounds — the function doesn't just stay bounded, it becomes insignificant or completely dominant in the limit.

⚡ Big vs Little — Key Difference
O(g) — Big O
Allows f/g to stay bounded (f/g → 5 is fine). f can grow at the same rate as g.

Test Your Knowledge

5 questions to check your understanding. Select an answer to reveal the explanation.

Question 1

Which notation describes an upper bound on a function's growth rate?

💡 Big O is the Upper Bound
f ∈ O(g) means f(n) ≤ c·g(n) for all n ≥ n₀. It provides a ceiling — the worst-case guarantee. Big Ω is the lower bound, and Θ requires both simultaneously.
Question 2

If f(n) = 5n² + 3n + 7, which is the correct Big O classification?

💡 Drop Constants and Lower Terms
As n → ∞, only the dominant term matters. 5n² dominates 3n and 7. We drop the constant 5, giving O(n²). Big O notation ignores multiplicative constants.
Question 3

What does lim(f/g) = 0 as n → ∞ indicate?

💡 Little o means the Ratio Vanishes
Little o (strict upper) requires lim f/g = 0. If f/g → 0, f is asymptotically negligible relative to g. Big O only requires lim f/g ≤ L < ∞.
Question 4

Place these in order from slowest to fastest growth:

n³, log n, 2ⁿ, n log n, n!

💡 Canonical Ordering
The full ordering is: O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!). Note n³ grows faster than n log n but slower than 2ⁿ.
Question 5

Is the following TRUE or FALSE? "If f ∈ O(g), then g ∈ Ω(f)"

💡 TRUE — O and Ω are Dual Notations
If f(n) ≤ c·g(n), then g(n) ≥ (1/c)·f(n). Big O and Ω are mirror images. f ∈ O(g) ⟺ g ∈ Ω(f). This is a fundamental symmetry of asymptotic analysis.
